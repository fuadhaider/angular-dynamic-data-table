Assumptions

Technical Stack

  • Angular 6

Development

  • Used the latest version of npm, Angular
  • No external Libraries

Design

  • Fully Responsive e.g. mobile
  • Minimal vanilla flavoured theme

UX/UI

  • Number of rows options are buttons rather than dropdown
  • Page navigation previous next
  • Filter triggers for every key stroke
  • Input field for jump navigation rather than list of few page links, ideal for large number of pages
  • Buttons disable as per usage e.g. previous button is disabled on first page

Features

  • Current and Total page number is fully reactive, even for filtered items
  • Table rows built in segments for fast loading
  • Clicking headers can sort columns by ascending and descending order
  • Filters by consecutive characters
  • Album 2 Id 34 can be filtered by entering 234
  • Setting item number or sort do not refresh the table to the beginning, e.g. if a user is on page 30 on item view 100 per page, changing to 50 per page will not take the user back to the beginning, table will reorganised accordingly in the background, user will stay on the same table segment

Under the bonnet

  • Single data fetch from server
  • Data manipulation do not call server, fully on client side
  • A service loads JSON file from url and sends it to component
  • Component subscribes JSON data show it in a table
  • Component also decorates and control the table and its options
  • Service and Components are destroyed to avoid app running in the background e.g. memory leaks
